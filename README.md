### 브랜치 관리 전략

- Git Flow를 사용하여 브랜치를 관리합니다.

- Release,Develop 브랜치는 Pull Request 후 merge를 진행합니다.

- 메인 브렌치인 Develop인 경우 리뷰와 PR 을 필수로 하는 깃 protrction이 설정되어 있습니다.

- Develop : 완전히 개발이 끝난 부분에 대해서만 Merge를 진행합니다.
- Others : 기능 개발을 진행할 때 사용합니다.
